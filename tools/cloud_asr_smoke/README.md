# Cloud ASR Smoke

This is a strictly manual QA gate for Moongate cloud precise recognition. It is
not part of CI because it uploads local media and may consume paid API quota.

The default smoke validates the direct-SRT path with `whisper-1`. Text-only
transcription models can be smoke-tested only when you provide a timed guide SRT;
the script then requests JSON text and aligns it locally onto that guide timeline.

## Inputs

Set the upload/cost gate and every required input explicitly:

```zsh
MOONGATE_CLOUD_ASR_QA_RUN=I_UNDERSTAND_THIS_UPLOADS_AUDIO_AND_MAY_COST_MONEY \
MOONGATE_CLOUD_ASR_API_KEY='<already-authorized api key>' \
MOONGATE_CLOUD_ASR_AUDIO=/absolute/path/to/short-sample.wav \
MOONGATE_CLOUD_ASR_LANGUAGE=ja \
zsh tools/cloud_asr_smoke/run-cloud-asr-smoke.sh
```

Optional inputs:

- `MOONGATE_CLOUD_ASR_BASE_URL`: defaults to `https://api.openai.com`.
- `MOONGATE_CLOUD_ASR_MODEL`: defaults to `whisper-1`. JSON-only models require
  `MOONGATE_CLOUD_ASR_GUIDE_SRT`.
- `MOONGATE_CLOUD_ASR_PROMPT`: prompt sent with the transcription request.
- `MOONGATE_CLOUD_ASR_GUIDE_SRT`: optional timed SRT generated by local ASR or a
  platform subtitle. Required for JSON-only transcription models such as
  `gpt-4o-transcribe`.
- `MOONGATE_CLOUD_ASR_OUTPUT`: output SRT path. If omitted, the script writes to
  a temp directory.
- `MOONGATE_CLOUD_ASR_WORKDIR`: response/body work directory.

Example for a JSON-only model with a local timing guide:

```zsh
MOONGATE_CLOUD_ASR_QA_RUN=I_UNDERSTAND_THIS_UPLOADS_AUDIO_AND_MAY_COST_MONEY \
MOONGATE_CLOUD_ASR_API_KEY='<already-authorized api key>' \
MOONGATE_CLOUD_ASR_AUDIO=/absolute/path/to/short-sample.wav \
MOONGATE_CLOUD_ASR_MODEL=gpt-4o-transcribe \
MOONGATE_CLOUD_ASR_GUIDE_SRT=/absolute/path/to/short-sample.local-asr.ja.srt \
MOONGATE_CLOUD_ASR_LANGUAGE=ja \
zsh tools/cloud_asr_smoke/run-cloud-asr-smoke.sh
```

## What It Proves

- The configured endpoint accepts Moongate's OpenAI-compatible
  `/v1/audio/transcriptions` multipart request.
- `response_format=srt` returns a non-empty SRT-like body for direct-SRT models.
- `response_format=json` can be converted to timed SRT when a guide SRT is
  supplied for local alignment.
- The manual QA path preserves the upload/cost confirmation gate.
- Non-direct-SRT models such as `gpt-4o-transcribe` stay blocked unless a guide
  SRT is explicitly provided.

## What It Does Not Prove

- It does not prove a paid end-to-end queue run for text-only transcription models.
- Alignment quality is only as good as the guide SRT timeline; use a short
  sample and inspect the output manually before trusting a new model.
- It does not prove translation, burn-in, or visual queue UI behavior.
- It does not store, print, or manage API keys; use only an already authorized
  test credential for the current QA run.
