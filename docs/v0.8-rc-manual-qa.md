## Known Release Limits

- Builds are ad-hoc / unsigned for local QA only.
- Apple notarization is intentionally not done.
- Windows real-hardware QA is not done; current validation is macOS-hosted .NET plus Parallels Windows 11 VM.
- Local ASR real-runtime QA requires user-provided `whisper-cli`, model, ffmpeg, and a local sample; normal CI/preflight does not download any runtime or model.

## Smoke Path

1. Open the app and confirm Settings opens as a separate window.
2. In Settings, confirm General, Subtitles & Translation, Local Speech Recognition, AI Services, Video & Output, Site Login, Components & Storage, and Updates & About panes are visible.
3. In Site Login, confirm "Clear this app's login data" is destructive/red and has confirmation.
4. In Updates & About, confirm the About section has one GitHub repository button and the version appears only in the update section.
5. Parse a video with platform/manual subtitles.
6. On the ready page, choose exactly one subtitle source and one subtitle output mode.
7. Start a normal non-ASR download and confirm progress starts as download progress rather than jumping ahead to later phases.
8. Parse a video with no usable subtitles; if local ASR is configured and ready, confirm Local Speech Recognition appears as a subtitle source.
9. For a completed download, use the queue waveform action to rerun local speech recognition, translation, and burn-in without re-downloading the video.
10. Download a 2160p/4K-capable video and confirm the selected 4K format is not silently constrained to 1080p.
11. Run a burn-in job and compare output size against source; large blow-ups should no longer happen on ordinary known-bitrate sources.
12. Let a batch finish and confirm completion notification and sound follow the app settings.

## API Translation Path

1. Reset onboarding only if needed for QA.
2. Choose a non-local translation provider in onboarding.
3. Confirm the same API configuration surface as Settings appears: base URL, API key, model, fetch models, and test connection.
4. In Settings > AI Services, confirm Anthropic-compatible and OpenAI-compatible model fetching works with the configured endpoint.

## Local ASR Settings Path

1. Open Settings > Local Speech Recognition.
2. Confirm the model list shows concise model size and capability text.
3. Confirm an installed model can be selected with "Use" and then shows "In use".
4. Confirm runtime path, model path, and model ID fields are behind Advanced.
5. Confirm model download is explicit; the app should not auto-download a runtime or model.
