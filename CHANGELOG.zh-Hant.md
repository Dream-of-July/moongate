# 更新日誌

[English](CHANGELOG.md) · [简体中文](CHANGELOG.zh-Hans.md) · **繁體中文**

本專案版本遵循語意化版本（major.minor.patch）。

## 未發布

Ready 頁改為語言優先的字幕選擇：你選語言而不是技術來源，下載後系統自動選用品質最高的來源。

### 新增 / 改進

- **語言優先 Ready 頁**：Ready 頁預設只展示一個由影片內容動態判斷出的推薦語言（依影片標題與可用字幕軌道決定——如日語 MV 推薦日語、英文訪談推薦英語、韓語 MV 推薦韓語），其他語言與來源細節收進「更多語言」展開區。
- **下載後自動定源**：選定語言後自動選用品質最高的來源。人工/官方字幕直接信任；平台自動字幕依可用性評估（語言匹配、cue 密度、覆蓋率、亂碼/重複——絕不比時序），品質差且本機識別可用時自動改用本機 Whisper 並在展開區標明實際來源；本機識別不可用時不阻塞，沿用自動字幕並給出「可啟用本機識別」的提示。
- **設定文案**：「增強模式」改為「AI 翻譯規劃」；本機語音識別說明改為「當平台字幕品質較差時自動補充識別」。

### 測試

- 新增語言推薦器與平台字幕品質門的跨端純邏輯測試（含「品質門絕不拿時序判 whisper」的防迴歸用例），以及佇列回退與 Ready 頁接線測試。評分/門檻常量統一進共享 fixture，兩端各自斷言。

## 0.7.6

0.7.6 是一次跨平台字幕分段與時間軸改進版本，重點優化英文與日文字幕在 YouTube 自動字幕、手動字幕和翻譯前清洗流程中的斷句、顯示時長和句間銜接。macOS 與 Windows 核心邏輯保持同構發行。

### 修復 / 改進

- **英文分段更像真實字幕**：新增弱語意邊界辨識，避免把 `the / and / which / to` 等連接詞切到難讀的位置；短回饋句、冒號/分號後的句間交接和長停頓顯示時長更穩定。
- **日文無空格字幕更穩**：對日文/CJK 字元級 timing 做單獨處理，減少自動字幕捲動 carry、短密字幕和人工多行字幕被拆成 blink cue 或拖尾過長的問題。
- **保留 VTT 詞級時間戳**：下載自動字幕時優先保留 VTT inline word timestamps，清洗階段使用來源片段時間而不是只依賴整條 cue 視窗。
- **跨平台同構分段器**：新增 Swift / C# 雙端 `SubtitleTimingPlanner`，集中管理可讀時長、弱邊界、短句和 source-anchored timing 規則。
- **評估工具落地**：新增 `tools/subtitle_timing_eval/`，用於用公開樣本、ASR/VTT 參考和離線 metrics 檢查分段時間軸；產生產物預設寫入 ignored artifacts。
- **工作進度更清晰**：佇列層同步工作級進度與 ETA，處理 VTT→SRT 正規化後再燒錄或輸出。

### 測試

- 新增 Swift / Windows 分段器、VTT 解析、字幕清洗和 queue 行為回歸測試。
- 新增 Python 離線評估工具單元測試，涵蓋 VTT word timing、弱邊界、manifest/status/runbook 和翻譯字幕 overlap gate。

## 0.7.5

0.7.5 是一次 Windows-only 熱修版本，重點修復 Windows 版設定視窗在開啟時因 WPF 唯讀繫結觸發 `RangeBase.Value`/`Run.Text` 初始化例外的問題，並補齊 Windows on ARM 虛擬機中的基礎建置、安裝程式和啟動煙霧測試驗證。macOS 最新發行面仍停留在 0.7.3。

### 修復 / 改進

- **Windows 設定視窗不再因更新區繫結崩潰**：設定頁更新區所有 `Updater.*` 繫結顯式使用單向繫結，避免 WPF 嘗試寫回唯讀狀態屬性。
- **更新區文字不再觸發 `Run.Text` 例外**：設定頁更新區域移除 inline `Run` 文字繫結，改為一般 `TextBlock` 組合顯示；點擊「設定」不應再彈出 `Set property 'System.Windows.Documents.Run.Text' threw an exception`。
- **進度值防護**：下載、解析和佇列 UI 進度統一過濾 `NaN`、無窮大和越界值，避免 WPF `RangeBase` 控制項收到非法值。
- **憑證錯誤提示更準確**：當 yt-dlp/系統網路堆疊因代理根憑證或系統根憑證不受信任而失敗時，錯誤文案會提示檢查 Windows 系統時間、根憑證和代理/VPN 憑證，而不是泛化成普通網路波動。
- **Windows 發行鏈路驗證**：在 Windows on ARM 虛擬機上驗證 `win-x64` 自包含發行產物、NSIS 安裝程式建置、暫存目錄安裝和啟動煙霧測試；目前仍透過 x64 模擬執行，不宣告原生 ARM64。

### 測試

- Windows `dotnet test windows/Moongate.Win.sln` 涵蓋 414 個核心測試與 1 個 WPF 設定視窗初始化煙霧測試。

## 0.7.3

0.7.3 是一次面向**發行加固**的版本：按外部程式碼審查，系統性收緊了更新、相依、登入憑證、Cookie 隔離與設定可靠性等發行前風險點，並修復了下載進度條卡在 100% 的體驗問題。

> 說明：本版改動均已通過單元測試與編譯驗證，但**尚未在真實 Windows/macOS 機器上端對端驗證**（安裝、覆寫更新、解除安裝、WebView2、登入、高 DPI 等）；標穩定版發行前仍需真機驗證。詳見 `docs/audit-progress.md`。

### 安全

- **API Token 不再以明文寫入磁碟**：macOS 用 Keychain、Windows 用 DPAPI 加密儲存；舊版 settings.json 裡的明文 Token 首次啟動交易式遷移（先寫安全儲存、成功才抹除明文，失敗不遺失）。
- **Cookie 按站點隔離**：登入 Cookie 改為按站點（YouTube / Bilibili）分檔案匯出與過濾，下載按 URL host 選用對應 jar，互不汙染；舊全域 cookies.txt 自動遷移。
- **相依完整性校驗**：Windows 受管相依改為隨 App 固定版本下載，並在安裝前校驗 SHA-256，不再直接追隨上游 `latest`。
- **登入清除更誠實**：Windows「清除全部登入」在 WebView2 資料刪不掉時如實提示「部分清除」並下次啟動補刪。

### 修復 / 改進

- **下載進度不再卡 100%**：修復 DASH 視訊/音訊分流下載時，視訊串流到 100% 後進度條被永久凍結（音訊下載+合併期間看似「沒在下載」）的問題。
- **Windows 更新安裝競態**：更新前檢查佇列、給 NSIS 傳 PID 等舊行程退出再覆寫、專用退出態、暫存目錄清理；SemVer 正確處理預發行。
- **相依視窗可取消**：首次相依下載可取消、可關閉視窗、顯示真實位元組/速度進度；「重新下載相依」改為先下後換（失敗不破壞現有可用環境）。
- **設定可靠性**：設定檔損毀改為備份+提示而非靜默回預設；登入/相依跳轉儲存失敗時保持設定視窗開啟不遺失草稿。
- **macOS 不再代管全域 Homebrew**：移除 App 內「解除安裝 Homebrew 相依」，避免誤解除安裝使用者為別的專案裝的套件；二進位定位支援自訂 HOMEBREW_PREFIX。
- **Windows 路徑/解析**：資料夾名規避保留裝置名（CON/NUL/COM1…）；多連結擷取統一為按 http(s) 錨點切分；記住上次下載選項。
- **安裝程式多語言**：NSIS 安裝程式支援簡中/英文/繁中，桌面捷徑改為可選；解除安裝可選刪除使用者資料（設定/憑證/Cookie + 相依快取）。

### 測試

- macOS `swift test`、Windows `dotnet test windows/Moongate.Win.sln` 全綠；新增涵蓋憑證遷移、Cookie 隔離、相依體檢/校驗、SemVer、進度合併、路徑淨化等的單元測試。

## 0.7.2

0.7.2 是一次面向字幕翻譯品質與發行面一致性的修正版本，重點處理自動產生字幕在時間軸、斷句和雜訊文字上的問題，讓下載後的字幕翻譯更少出現跨句誤譯、重複標籤和難以追蹤的回退行為。

### 新增

- **ASR 字幕重分段**：對自動產生字幕增加可設定的重分段前處理，把過短、過碎或明顯斷錯的 cue 合併成更適合翻譯的語意片段，減少逐詞字幕導致的上下文遺失。
- **更穩的翻譯回退記錄**：當 ASR 重分段或增強模式不可用時，macOS 與 Windows 核心會保留更清楚的 fallback 日誌，方便判斷是設定、輸入字幕還是模型回應導致效果變化。

### 修復 / 改進

- **字幕清洗更嚴格**：修復 SRT 清洗中的跳脫與標籤處理問題，降低 HTML/ASS 標籤、重複雜訊和異常 cue 進入翻譯提示詞的機率。
- **ASR 判斷更保守**：強化自動字幕啟發式判斷，避免把正常人工字幕誤當作 ASR 字幕進行過度重分段。
- **文案降噪**：把「智慧翻譯提示詞」統一改名為「增強模式」，更貼近使用者實際開關含義，也避免介面解釋模型內部實作。
- **發行面收口到 0.7.2**：macOS/Windows 預設建置版本、安裝包命名、GitHub Actions 預設參數、README、Windows 文件與發行面測試統一更新到 0.7.2。

### 測試

- 新增和更新 macOS / Windows 字幕清洗、ASR 重分段、翻譯設定遷移、回退路徑和本地化文案測試。
- 更新發行面測試，涵蓋 macOS 建置指令碼、Sparkle ZIP/appcast 指令碼、Windows 安裝程式、GitHub Actions 預設版本與文件產物名。

## 0.7.0

本版本完成桌面端 0.7 多語言與首啟體驗迭代：macOS / Windows 面向英語、簡體中文、繁體中文使用者可用，翻譯目標語言可獨立選擇，並補齊更清晰的無 API 使用路徑。

### 新增

- **三語 UI 與設定**：macOS 與 Windows 支援 English / 简体中文 / 繁體中文介面語言，設定中可即時切換；翻譯目標語言獨立於介面語言，可選擇簡體中文、繁體中文或 English。

- **更多站點辨識**：TikTok、抖音、小紅書及常見短連結網域會優先走 yt-dlp 原生擷取路徑，失敗時保留登入、風控、地區或平台限制的誠實提示，不偽裝成通用網頁偵測。

### 修復 / 改進

- macOS 主視窗、設定、佇列、登入、摘要、相依、更新與關閉確認等高頻介面文案接入執行階段本地化。

- **macOS 自更新遷移到 Sparkle**：App 內更新不再下載 DMG 或 PKG 後自行處理安裝；現在使用 Sparkle 2 原生更新視窗，從 GitHub Pages appcast 發現更新，下載 GitHub Release 裡的 `Moongate-macOS-v0.7.0.zip`，通過 Sparkle EdDSA 簽章校驗後取代 App。
- **免 Developer ID 的低成本鏈路**：新增 Sparkle ZIP 與 appcast 發行指令碼，私鑰儲存到本機 Keychain，儲存庫只儲存公開公鑰。該路線解決 App 內更新與防竄改，但不偽裝成 Gatekeeper 級正式發行體驗。
- **未完成佇列保護**：佇列裡還有未完成工作時，設定頁會先阻止手動檢查更新，提示先完成或取消工作，避免更新重啟與下載 / 壓製流程互相衝突。
- **發行資產邊界調整**：App 內自動更新資產改為 `Moongate-macOS-v0.7.0.zip` + `docs/appcast.xml`；DMG 僅保留為手動拖曳安裝兜底，PKG 保留為未來 Developer ID Installer 鏈路備用。
- 發行指令碼、Windows 安裝程式、GitHub Actions 預設版本和文件範例更新到 0.7.0。

## 0.6.1

修復 macOS 自動更新「能下載卻裝不上」的問題。

### 新增

- **記住上次下載選項**：下載影片時選擇的「字幕處理方式 / 字幕語言 / 輸出格式 / 是否 HDR」會被記住，下一次下載自動沿用上次的選擇（例如「英語字幕 - 翻譯並壓製」）。字幕按語言代碼記憶，會在下個影片實際可用的字幕裡自動比對勾選。

### 修復 / 改進

- **修復 macOS 自更新靜默失敗**：下載新版 DMG 後，舊版本在退出時會過早卸載安裝映像，導致取代指令碼從映像複製新 App 時來源已消失（`ditto: Cannot get the real path for source`），`/Applications/月之门.app` 始終沒被取代、重啟仍是舊版。現在映像的卸載改由取代指令碼在複製完成後負責，更新可穩定完成。
  - 注意：該缺陷位於舊版本自身的更新器中，已安裝的 0.5.0 / 0.6.0 仍需手動下載本版本一次；自 0.6.1 起，後續版本即可正常自動更新。
- **自更新安裝目錄不可寫時給出明確錯誤**：取代指令碼在 App 退出後執行，失敗無法回傳介面。現在退出前先檢查安裝目錄可寫性（如非系統管理員帳戶下的 /Applications），不可寫時直接提示用「系統管理員帳戶執行或到 GitHub 手動下載」，不再靜默失敗。
- **修復 OpenAI 相容引擎「測試連線 / 擷取模型」報錯**：測試連線此前固定呼叫 OpenAI 私有的 `/v1/responses` 端點，而多數「OpenAI 相容」服務（Azure、DeepSeek、OpenRouter、Ollama、本機推理服務等）只實作 `/v1/chat/completions`，導致設定能儲存、測試卻失敗。現在 OpenAI 協定統一改用通用的 `/v1/chat/completions`；同時擷取模型清單時不再向非 Anthropic 服務傳送 Anthropic 專有請求標頭，避免嚴格閘道因未知標頭拒絕。（桌面與行動端同步修復）

### 測試

- 更新 macOS 自更新指令碼產生測試：校驗映像卸載發生在複製之後、且由指令碼負責。
- 通過 SwiftPM 全量建置與 UpdateChecker 測試；並在真實 UDZO（中文卷名）映像上重現缺陷並驗證修復。

## 0.6.0

本版本繼續推進燒錄 / 轉檔效能優化：在可安全使用硬體路徑的環節盡量呼叫系統最佳媒體引擎，並把下載後轉檔階段的進度回饋補齊。

### 新增

- **硬體加速規劃器**：macOS 轉檔路徑增加 VideoToolbox 輸入硬體加速規劃；Windows 轉檔路徑增加 NVENC / Intel Quick Sync / AMD AMF 對應的輸入硬體加速規劃。無濾鏡轉檔可更完整地把解碼與編碼交給硬體媒體鏈路。
- **轉檔百分比提示**：下載後的轉檔階段現在顯示「轉檔中 X%」，進度未知時顯示「轉檔中…」，避免和一般後處理混在一起。

### 修復 / 改進

- **壓製 / 轉檔文案降噪**：設定、文件與本地化提示不再顯式標註 CPU 回退，只提示遇到相容性問題時耗時可能比預計更長。
- **相容路徑更明確**：HDR 轉 H.264 的 tonemap 等必須經過軟體濾鏡的路徑會保留相容性提示，同時避免錯誤注入硬體輸入加速參數。
- **Windows 佇列狀態對齊**：Windows 端補齊下載後處理類型，轉檔階段與 macOS 一樣展示專門的進度文字。
- **更新檢查錯誤文案修正**：GitHub 限流、網路失敗等更新檢查錯誤不再誤顯示為「解析影片資訊失敗」。

### 測試

- 新增 macOS / Windows 轉檔規劃、硬體輸入加速參數、HDR 相容路徑、轉檔進度 UI、本地化文案與更新檢查錯誤邊界測試。
- 完整驗證通過 SwiftPM 全量測試、Windows 核心庫全量測試、Windows App 建置與發行面字串檢查。

## 0.5.0

本版本聚焦燒錄 / 轉檔的效能與畫質：把編碼交給 Apple Silicon 的硬體媒體引擎（VideoToolbox），並修復一批燒錄、設定與互動問題。

### 新增

- **硬體編碼加速（VideoToolbox）**：燒錄與轉檔可交給 Mac 的硬體媒體引擎，4K 編碼更快、更省電。設定 →「燒錄與轉檔」新增「編碼方式」三檔：自動（硬體優先，推薦）/ 硬體加速 / 軟體（畫質最高，較慢）；遇到來源檔案或系統相容性問題時，實際耗時可能比預計更長。HDR 來源走硬體 `hevc_videotoolbox` main10 並透傳 HDR10 色彩中繼資料（mastering-display / max-cll 保留），與軟體 libx265 路徑畫質中繼資料等價。
- **自適應壓製並行**：硬體後端下自動比設定值多並行一路壓製（上限 4），整體吞吐更高；相容路徑維持原值，避免多路工作互相拖慢。
- **燒錄編碼可選**：設定 →「燒錄與轉檔」新增「燒錄編碼」：跟隨來源（HEVC 來源保 HEVC、保 HDR、體積更小）/ 始終 H.264（相容最好，幾乎所有裝置與網頁都能播）。

### 修復 / 改進

- **修復轉檔無進度條**：下載後轉檔階段現在即時顯示進度（此前缺 `-progress` 輸出，進度條不動）。
- **修復 HEVC 來源燒字幕被降級成 H.264**：跟隨來源編碼時 HEVC 4K 來源燒完字幕仍是 HEVC，不再悄悄降到 H.264、掉位元率掉解析度。
- **修復「縮放到 1080p」關閉後不生效**：關閉該開關現在能持久化（此前因 `maxBurnHeight` 為空被序列化省略、重新載入又當成預設 1080，導致關閉存不住，仍壓成 1080）。
- **AI 摘要高光改為海浪效果**：計算中的「正在理解影片內容」文案改為逐字連續起伏的海浪高光（波峰沿文字捲過、字元上浮提亮，掃完留一段平靜間隔再湧起），取代此前整行一道的線性流光；尊重「減弱動態效果」。
- **AI 翻譯 / AI 摘要的「單獨設定」補齊擷取模型與測試連線**：與主「AI 設定」一致，可先填位址+憑證、擷取伺服器真實模型清單再選，並測試連線。

### 測試

- 新增編碼器選擇矩陣單元測試（硬體 / 軟體 × 來源編碼 × HDR × 強制 H.264，含硬體不可用回退）、轉檔硬體路徑與 HDR main10 中繼資料、編碼後端與燒錄編碼設定的持久化、硬體後端有效並行數等。
- ffmpeg 端對端驗證三條關鍵路徑：SDR HEVC 保 HEVC、強制 H.264、HDR 硬體 main10 保留 PQ 中繼資料。

## 0.4.0

產品於本版本正式定名為 **月之門**。這是一次大更新，集中在 macOS 端的 AI 能力、HDR 支援與自助更新。

### 新增

- **產品定名「月之門」**：應用名、視窗標題、Dock、關於面板、安裝包均為「月之門」；Bundle 標識 `com.moongate.app`，安裝為 `/Applications/月之门.app`，Tahoe 分層圖示對齊。
- **AI 影片摘要**：選片頁一鍵用 AI 概述影片內容（資料來源優先字幕、回退影片簡介），下載前確認不下錯；計算時為 Apple Intelligence 風格的跑馬燈流光邊框，完成後結果展開動畫。
- **HDR / 杜比視界下載**：解析辨識每檔畫質的 HDR 來源，選片頁提供「HDR」開關；HDR 預設 mkv 封裝保真。
- **下載後轉檔 / remux**：選片頁可選輸出格式（保持來源 / MP4 H.264 / MP4 H.265 / MKV）。同編碼換容器走 remux（無損秒級），跨編碼轉檔；HDR 轉 H.265 用 libx265 10-bit 保留 HDR，轉 H.264 會 tonemap 成 SDR 並提示。
- **HDR 保真燒字幕**：在 HDR 畫面上燒錄中文字幕（libx265 10-bit + HDR10 中繼資料透傳），字幕為 SDR 顏色；libx265 不可用時回退 tonemap 成 SDR。
- **統一「AI 設定」**：翻譯與摘要共用一份預設 AI 設定，各自可「跟隨預設」或單獨設定；新增 Apple Translation（低延遲/高保真）、Apple Intelligence 本機、PCC/Cloud Pro 引擎（按系統能力誠實判斷可用性）。
- **未登入引導**：偵測到 YouTube / B 站等需要登入導致失敗時，失敗頁提供「去登入」按鈕，彈站點登入頁儲存 cookies 後重試。
- **遠端更新**：設定 → 更新可檢查並全自動下載安裝新版（來源 GitHub Releases，校驗下載位址與 bundle 標識，取代後自動重啟），失敗有「去 GitHub 下載」兜底。
- **相依元件管理升級**：相依區摺疊化（全就緒摺一行、有缺失自動展開），新增「刪除相依」（紅色警示 + 二次確認），安裝/解除安裝逐項顯示旋轉→綠勾的遞進狀態。
